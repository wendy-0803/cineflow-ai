# 作品集与面试包装

## 项目名称

AI 短片脚本生成与分镜规划工具

英文名可用：CineFlow AI

## 推荐项目描述

我独立设计并实现了一个面向短视频创作者的 AI 短片脚本生成与分镜规划工具。产品支持从一句创意生成故事大纲、三幕结构、角色卡、分场景脚本、镜头分镜、画面提示词、旁白字幕和导出方案。项目重点不是直接生成视频，而是解决创作者在使用即梦、可灵、Runway、纳米大片等视频工具前的脚本规划和提示词组织问题。

## 简历写法

### 项目经历

**CineFlow AI｜AI 短片脚本生成与分镜规划工具｜独立产品项目**

- 面向短视频创作者设计 AI 前置创作工作流，支持从一句创意生成故事结构、角色卡、分场景脚本、镜头分镜和多平台视频提示词。
- 拆解即梦、纳米大片、可灵、Runway 等 AI 视频工具的创作流程，定位“脚本与分镜前置规划”作为 MVP 切入点，避免直接与视频生成模型能力竞争。
- 设计多阶段 Prompt 流水线，将生成过程拆为创意理解、故事结构、角色设定、场景拆分、镜头脚本、提示词优化和一致性检查，提高输出稳定性和可编辑性。
- 搭建 Web MVP，支持项目创建、结构化生成、分镜编辑、局部重生成、Markdown / JSON / PDF 导出和历史项目管理。
- 通过 Demo 测试记录生成耗时、用户编辑字段和提示词可用性，沉淀项目复盘与后续迭代计划。

## 面试讲述结构

### 1. 为什么做

AI 视频工具越来越强，但创作者真正卡住的地方不只是“生成一段视频”，而是从创意到脚本、分镜、旁白、字幕、提示词这一整段规划过程。很多工具对单条提示词友好，但多镜头故事的连续性、角色一致性、镜头节奏仍然需要用户自己整理。

### 2. 用户是谁

核心用户是短视频创作者、内容运营、AI 视频工具使用者。他们通常有创意，但缺少脚本和分镜能力，也不熟悉如何把创意拆成适合视频工具生成的提示词。

### 3. 做了什么

我做的是一个 AI 短片创作前置工作台：用户输入题材、风格、时长、平台、受众、主角和情绪后，系统生成故事梗概、三幕结构、角色卡、分场景脚本和镜头分镜。每个镜头包含画面描述、镜头运动、台词旁白、字幕、音效、时长和 AI 视频提示词。

### 4. 关键产品判断

我没有一开始做视频生成，因为这会变成模型能力竞争，难以体现产品价值。我选择做“生成前的规划层”，一方面更适合个人 MVP，另一方面能真实连接即梦、可灵、Runway、纳米大片等工具。

### 5. 技术实现

前端使用 Next.js / React，后端使用 API Routes，AI 生成使用结构化 Prompt 流水线。生成不是一个大 Prompt，而是拆成创意理解、故事结构、角色设定、场景拆分、镜头脚本、视频提示词优化和一致性检查。结果以 JSON 保存，前端渲染成可编辑分镜表，并支持导出。

### 6. 难点

- 如何让 AI 输出稳定结构化内容。
- 如何保持多镜头角色一致性。
- 如何让局部重生成不破坏全局故事。
- 如何设计既适合创作者理解，又适合视频工具投喂的输出格式。

### 7. 数据与反馈

MVP 阶段我会记录：

- 从创意到完整分镜的生成耗时。
- 用户最常编辑哪些字段。
- 用户是否愿意复制提示词到下游工具。
- 生成结果的可用性评分。
- 哪些题材最容易生成高质量结果。

### 8. 后续迭代

- 支持不同视频工具的提示词模板。
- 增加参考图和角色一致性素材管理。
- 增加脚本质量评分。
- 增加 Demo 模板库。
- 接入真实视频工具 API 或半自动跳转投喂。

## 面试中不要这么说

- 不要说这是不存在公司的实习经历。
- 不要夸大已有用户数据。
- 不要说已经接入所有视频生成工具，除非真的接入。
- 不要把项目说成大而全的视频生成平台。

## 更稳的说法

这是我为了准备 AI 产品岗位做的独立产品实战。我重点验证的是 AI 视频创作流程里“前置规划”这个环节的需求，并通过 MVP 展示从用户洞察、PRD、Prompt 流水线、技术实现到 Demo 复盘的完整能力。
