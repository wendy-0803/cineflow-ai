# 从 0 到可面试项目路线图

## 阶段 0: 项目定位

目标：明确这是一个真实 AI 产品项目，而不是虚构实习。

产出：

- 项目名称
- 一句话定位
- 目标用户
- MVP 范围
- 简历描述初稿

状态：已完成初稿。

## 阶段 1: 产品调研

目标：证明你不是凭空想功能，而是基于真实工具和用户流程做判断。

任务：

- 体验即梦、纳米大片、可灵、Runway。
- 每个产品记录创作入口、脚本能力、分镜能力、提示词控制、版本管理。
- 总结 3 个用户痛点。
- 输出竞品调研表。

产出：

- `docs/research/competitor-research.md`
- 调研截图
- 竞品流程图

## 阶段 2: PRD

目标：把项目从想法变成可执行产品。

任务：

- 明确目标用户。
- 明确核心场景。
- 定义 P0 / P1 / P2。
- 画页面流程。
- 定义输出字段。

产出：

- `docs/prd/PRD.md`
- 页面流程图
- 功能优先级表

## 阶段 3: 原型

目标：先验证工作流，不急着写完整代码。

页面：

- 工作台
- 新建项目
- 脚本生成
- 分镜编辑
- 导出
- 历史项目

产出：

- 低保真原型
- 页面截图
- 用户流程说明

## 阶段 4: Web MVP

目标：做出可点击、可生成、可导出的 Demo。

任务：

- 搭建 Next.js 项目。
- 完成页面 UI。
- 使用 mock 数据跑通流程。
- 接入 AI API。
- 实现导出。
- 实现保存项目。

产出：

- 本地可运行项目
- 部署链接
- GitHub 仓库

## 阶段 5: Prompt 流水线

目标：让生成质量足够稳定，并能在面试中讲清楚。

任务：

- 创意理解 Prompt
- 故事结构 Prompt
- 角色设定 Prompt
- 场景拆分 Prompt
- 镜头脚本 Prompt
- 视频提示词优化 Prompt
- 一致性检查 Prompt

产出：

- `docs/prompts/prompt-pipeline.md`
- 结构化 JSON 示例
- 失败案例和优化记录

## 阶段 6: 反馈与复盘

目标：让项目不只是 Demo，而是有产品迭代意识。

任务：

- 找 3 到 5 个同学或创作者试用。
- 记录他们是否看得懂、愿不愿意用、哪里要改。
- 记录常见修改字段。
- 总结下一版迭代。

产出：

- 用户反馈表
- 项目复盘
- Demo 视频

## 阶段 7: 面试材料

目标：形成可以直接讲的项目经历。

产出：

- 简历项目描述
- 1 分钟项目介绍
- 3 分钟深挖讲述
- PRD 截图
- 原型截图
- Demo 链接
- GitHub 链接
- 项目复盘页
