## 1. 产品定位

CineFlow AI 是一个面向短视频创作者的 AI 前置创作工具。它帮助用户从一句创意出发，生成故事梗概、三幕结构、角色设定、分场景脚本、镜头分镜、画面提示词、旁白字幕和导出文件。

产品不在 MVP 阶段直接生成视频，而是聚焦解决“从想法到可投喂视频工具的创作方案”这一段断层。

## 2. 目标用户

### 核心用户

- 短视频创作者：需要快速产出剧情短片、广告短片、情绪短片、知识类短视频。
- AI 视频工具使用者：已经使用或准备使用即梦、可灵、Runway、纳米大片等工具，但不知道如何写稳定、连贯、可执行的提示词。
- 内容运营 / 产品实习生 / 个人创作者：需要低成本完成视频创意方案和分镜脚本。

### 用户痛点

- 只有一个创意，不知道怎么扩展成完整故事。
- AI 视频工具擅长生成片段，但不擅长保持剧情连续性和角色一致性。
- 分镜、旁白、字幕、音效、镜头运动需要手动整理，耗时且格式混乱。
- 不同视频工具对提示词格式要求不同，用户需要重复改写。
- 项目过程中版本混乱，修改某一幕会影响整体一致性。

## 3. 产品目标

### MVP 目标

- 用户能输入短片创作参数。
- 系统能生成结构化短片方案。
- 用户能编辑某一幕或某个镜头，并局部重新生成。
- 用户能导出 Markdown、JSON、PDF。
- 用户能保存项目和查看历史版本。

### MVP 闭环

第一版必须跑通：

1. 用户输入一句创意和几个参数。
2. 系统生成故事方案。
3. 系统生成角色卡和分镜表。
4. 用户编辑一个镜头或局部重生成。
5. 用户复制单镜头提示词。
6. 用户导出 Markdown 或 JSON。

### 非目标

- MVP 不做真实视频生成。
- MVP 不做多人协作。
- MVP 不做完整素材资产管理。
- MVP 不追求复杂社交和社区能力。

## 4. 核心输入

用户新建项目时填写：

- 创意一句话：例如“一个失眠的女孩在雨夜遇见了未来的自己”。
- 题材：剧情、科幻、悬疑、治愈、广告、知识、纪录感。
- 风格：电影感、赛博朋克、日系清新、国风、水墨、纪录片、广告大片。
- 时长：15 秒、30 秒、60 秒、90 秒、3 分钟。
- 平台：抖音、小红书、B 站、视频号、YouTube Shorts、TikTok。
- 目标受众：大学生、职场新人、年轻女性、AI 爱好者等。
- 主角设定：可选。
- 情绪基调：温暖、紧张、荒诞、孤独、热血、治愈。
- 生成语言：中文 / 英文。

## 5. 核心输出

### 5.1 故事方案

- 标题
- 一句话卖点
- 故事梗概
- 三幕结构
- 情绪曲线
- 核心视觉符号

### 5.2 角色卡

- 角色名
- 年龄 / 身份
- 外貌特征
- 性格与动机
- 视觉一致性提示词
- 声音 / 语气

### 5.3 分场景脚本

- 场景编号
- 场景名称
- 场景目的
- 地点 / 时间
- 剧情描述
- 角色行动
- 旁白 / 台词

### 5.4 镜头分镜

每个镜头包含：

- 镜头编号
- 所属场景
- 时长
- 景别：远景、全景、中景、近景、特写。
- 镜头运动：推、拉、摇、移、跟拍、手持、固定。
- 画面描述
- 角色动作
- 台词 / 旁白
- 字幕
- 音效 / 音乐建议
- AI 视频提示词
- 负向提示词
- 可投喂工具建议：即梦 / 可灵 / Runway / 纳米大片。

## 6. 页面流程

### 首页 / 工作台

- 展示历史项目。
- 支持新建短片项目。
- 展示最近导出的文件。

### 新建项目页

- 输入创意与参数。
- 选择风格、平台、时长、受众。
- 点击生成。

### 脚本生成页

- 展示故事梗概、三幕结构、角色卡、分场景脚本。
- 支持重新生成整体方案。
- 支持编辑单个模块。

### 分镜编辑页

- 以表格或卡片形式展示镜头列表。
- 每个镜头可编辑画面、运动、台词、字幕、提示词。
- 支持局部重生成某个场景或某个镜头。
- 支持复制单条提示词。

### 导出页

- 支持导出 Markdown。
- 支持导出 JSON。
- 支持导出 PDF。
- 展示不同视频工具的投喂建议。

### 历史项目页

- 项目列表。
- 项目状态：草稿、已生成、已导出。
- 版本历史。

## 7. MVP 功能优先级

### P0

- 新建项目。
- 输入创意参数。
- AI 生成故事方案、角色卡、分场景脚本、镜头分镜。
- 分镜可编辑。
- Markdown / JSON 导出。

### P1

- 局部重生成某一幕 / 某个镜头。
- PDF 导出。
- 项目保存。
- 版本历史。

### P2

- 多工具提示词格式适配。
- Demo 项目模板。
- 用户反馈收集。
- 提示词质量评分。

## 8. 成功指标

MVP 阶段可以手动记录：

- 从输入创意到生成完整分镜的耗时。
- 用户是否能理解输出结构。
- 用户是否愿意复制提示词到视频工具。
- 单个项目平均编辑次数。
- 用户最常修改的字段。
- 用户主观评分：故事可用性、提示词可用性、导出可用性。

## 9. 面试可讲亮点

- 没有直接做“AI 视频生成”，而是选择更适合 MVP 的前置规划环节。
- 把一个大模型生成任务拆成多阶段 Prompt 流水线，提高稳定性和可控性。
- 输出结构贴合真实创作者工作流，而不是只生成一段散文。
- 支持局部重生成，降低用户修改成本。
- 用导出能力连接即梦、可灵、Runway、纳米大片等下游工具。
