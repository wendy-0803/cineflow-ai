# CineFlow AI

AI 短片脚本生成与分镜规划工具。

这个项目定位为一个真实的 AI 产品项目 / 独立产品实战，而不是虚构公司实习。目标是做出一个可演示、可复盘、可被面试追问的产品：面向短视频创作者，支持从一句创意生成故事大纲、角色设定、分场景脚本、镜头分镜、画面提示词、旁白字幕与导出方案。

## 项目目标

- 做出一个可用 MVP：用户能新建短片项目，输入创意参数，生成脚本与分镜，并导出 Markdown / JSON / PDF。
- 沉淀完整产品材料：PRD、竞品调研、用户流程、Prompt 流水线、技术方案、Demo 记录、项目复盘。
- 形成面试表达：能讲清楚目标用户、痛点判断、功能取舍、技术实现、数据反馈与迭代方向。

## 当前目录

- `docs/prd/`: 产品需求文档与页面流程。
- `docs/research/`: 竞品调研、用户痛点、访谈记录。
- `docs/tech/`: 技术方案、架构设计、数据结构。
- `docs/prompts/`: Prompt 流水线、结构化输出规范。
- `docs/portfolio/`: 作品集、面试讲述稿、项目复盘。
- `app/`: Next.js App Router 版本产品代码。
- `preview/`: 无依赖静态预览版，可直接本地打开。
- `assets/`: 原型截图、Demo 图、素材。
- `notes/`: 日常想法、会议记录、反馈记录。

## 本地预览

当前已经有 mock 数据版 MVP。

如果只是想马上看页面，运行：

```powershell
.\start-preview.ps1
```

或直接打开：

`preview/index.html`

如果网络可以安装 npm 依赖，运行 Next.js 版本：

```powershell
.\start-next.ps1
```

如果你想把 Next.js 放到后台运行并把日志写入文件，运行：

```powershell
.\start-next-background.ps1
```

停止后台进程：

```powershell
.\stop-next.ps1
```

如果你想跑生产构建检查，运行：

```powershell
.\build-next.ps1
```

## 推荐完成顺序

1. 完成产品调研：拆解即梦、纳米大片、可灵、Runway 的创作流程。
2. 完成 PRD：明确 MVP 范围、页面流程和生成结果结构。
3. 做低保真原型：先用文档或 Figma 梳理页面，不急着写代码。
4. 搭建 Web MVP：Next.js + Tailwind CSS + shadcn/ui + SQLite/Supabase + OpenAI API。
5. 做 Prompt 流水线：创意理解、故事结构、角色设定、场景拆分、镜头脚本、视频提示词优化、一致性检查。
6. 加入项目保存、局部重生成、导出能力。
7. 录 Demo、写复盘、整理作品集。

## 面试包装方式

推荐说法：

> 我做了一个面向短视频创作者的 AI 短片脚本生成与分镜规划工具。它不直接做视频生成，而是解决创作者在进入即梦、可灵、Runway、纳米大片之前的前置规划问题：把一句创意拆成可执行的故事结构、角色卡、镜头脚本和视频生成提示词。项目中我负责从用户流程调研、PRD、Prompt 流水线设计到 Web MVP 实现和 Demo 复盘。

不要说：

> 我在某某公司实习做了这个。

这个项目的优势是：真实、可展示、可追问、能体现 AI 产品岗需要的产品判断和落地能力。
